# 🌌 原点分化双轨映射系统 & LLM Wiki (Harness Edition)

> **核心哲学**：同源双轨，基底分化。将 Obsidian 视作 IDE，将知识库视作由 AI 托管的代码库。本系统底层遵循“空间编码 (Space Coding)”理念，以**定义边界与状态审计**为核心操作。

---

## 🛠️ skill(操作)
将 AI 视作维护知识库的“程序员”。操作时必须严格遵循 ReAct 框架的先后逻辑（先 Action 后 Observation）。系统的底层运转完全依赖 `.claude/skills` 目录下的 10 个核心原子机能：

* **`ingest`**：**摄入解析**。处理 `00_Inbox` 的生语料，将其作为系统运行的原始燃料，进行初始解析与打标。
* **`query`**：**跨域检索**。穿透 4FS 架构，基于 `02_Wiki` 的主观认知与 `01_Materials` 的客观语料进行综合逻辑问答。
* **`search`**：**全库搜索**。底层文本定位工具，快速检索 Markdown 库中的特定关键字、标签或代码片段。
* **`defuddle`**：**逻辑排障**。分析并理清复杂概念，消除笔记间的逻辑冲突、死链或信息冗余。
* **`json-canvas`**：**可视化图谱处理**。解析和生成 Obsidian 专属的 `.canvas` 文件，用于推演技术架构（如 LangGraph 工作流的可视化推演）。
* **`lint`**：**状态与格式审计**。自动规范 Markdown 格式，清理无效元数据，确保“空间编码”状态下的数据绝对干净与统一。
* **`migrate`**：**流转迁移**。执行文件在 4FS 各目录间的物理移动（例如将提纯后的认知从 Inbox 移入 Wiki），严守系统边界定义。
* **`obsidian-bases`**：**底座维护**。管理 Obsidian 的核心数据结构，维护系统运行的底层索引基础。
* **`obsidian-cli`**：**终端桥接**。在本地终端环境执行 Obsidian 相关的命令行交互，实现系统外部到内部的自动化映射。
* **`obsidian-markdown`**：**特有语法解析**。针对 Obsidian 特有的 Markdown 方言（如双链、Dataview 查询块、Callout）进行深度解析与重构。

## 🔌 插件(可视化与技巧)
为确保系统在本地 Obsidian 中顺畅运转，需安装并启用以下核心第三方插件组：

* **AidenLx's Folder Note**：为文件夹创建同名文档（Folder Note），便于赋予目录以元数据与说明。
* **BRAT**：用于快速安装、测试与更新尚未上架社区的先锋/测试版插件。
* **Claudian**：核心 AI 调度中枢，将本地知识库与大模型能力无缝缝合。
* **Commander**：UI 定制神器，将高频脚本或宏命令直接挂载到侧边栏或文件菜单。
* **Custom File Explorer sorting**：自定义排序逻辑，雷打不动地维护 4FS 系统架构的展现优先级。
* **Dataview**：数据库级别的查询引擎，驱动 `03_Action` 中的 Dashboard 看板实现任务自动化聚合。
* **Local Images Plus**：自动化图片资产管理，确保从外部粘贴的图片自动收归至 `99_System/04_Assets` 并保持引用正确。
* **Marp**：快速导出化，将 `02_Wiki` 中的知识点利用纯 Markdown 语法一键渲染为演示幻灯片。
* **Number Headings**：文档长篇结构化利器，自动为层级标题分配编号。
* **Quiet Outline**：轻量级且无干扰的大纲目录视图，加速长篇概念文档的阅读与跳转。
* **Style Settings**：细粒度的界面深度定制，调整颜色映射以匹配个人视觉工作流。
* **Templater**：极其强大的模板引擎，在新建各区域文件时自动注入标准化格式与必要 Metadata。
* **Waypoint**：自动化生成并维护文件夹内的文件结构树，是构建局域索引的得力助手。

## 💻 本地环境
要完整激活本系统的“AI 托管”及外部拓展能力，需在 macOS 宿主机配置以下工具链：

1. **`qmd` (必备核心检索)**：专为本系统及插件提供底层检索支撑的命令行工具，实现知识库内 Markdown 文件的极速查找与精准上下文提取。
2. **`uv` (环境隔离)**：全库所有外部 Python 脚本（如 SkipClass 视频转录等）的唯一包管理工具。绝不污染系统级环境，确保即开即用、光速执行。
3. **`tmux` (并发支持)**：终端会话复用器。用于在后台静默挂载 `langgraph-api` 服务或本地自动化轮询脚本，保证断网或关掉终端后服务仍旧存活。
4. **Node.js/npm**：部分 Obsidian 高级插件、Marp 渲染引擎或本地全栈脚手架开发所需的必备运行时环境。

# 核心元文件维护规范

1.  **Index**: 必须体现知识的层级与交叉引用，而非简单的文件列表。
2.  **Glossary**: 词条必须具有“单一事实来源”，避免在多处笔记中重复定义。
3.  **Log**: 格式为 `| 日期 | 操作 | 源文件 | 影响节点 |`。
