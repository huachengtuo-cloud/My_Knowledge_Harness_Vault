# lint 技能 (系统健康与审计)

## 核心职责
防止知识库出现“熵增”，确保所有的信息流动符合双轨系统的规范。

## 审计流 (Audit Pipeline)

### 1. Frontmatter 校验
扫描 `02_Wiki_知识网络/` 下的所有笔记：
- 检查是否缺少 `noteType`、`infoValue`、`infoDensity` 等必要属性。
- 检查 `author` 是否统一为“农民伯伯”。
- 汇总不符合规范的文件列表，并提示用户是否自动修复。

### 2. 孤岛与死链检测
- **孤儿页面 (Orphans)**：找出 `02_Wiki` 中没有任何其他笔记指向它，且未登记在 `00_Index_全局索引.md` 中的页面。
- **死链 (Broken Links)**：找出使用 `[[ ]]` 引用但物理文件不存在的链接。

### 3. 行动中枢同步校验
- 扫描 `03_Action_执行空间/`：
- 找出状态已经是 `[x]` (已完成) 但超过 7 天未归档的任务，建议用户将其移除看板或转化为《实践结果》沉淀到 Wiki。

### 4. 输出报告
生成一份体检报告输出给用户，列出具体需要修复的文件路径及建议操作。
