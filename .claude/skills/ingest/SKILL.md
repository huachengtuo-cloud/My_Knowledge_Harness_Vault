# ingest 技能 (双轨分化版)

## 核心工作流：Inbox Zero 与基底分化

你正在维护一个名为 **My_Knowledge_Harness_Vault** 的知识库。
你的任务是将 `00_Inbox_原点` 的混沌信息，分化为**静态知识**和**动态任务**，并维护大盘中枢。

## 触发逻辑

1. **用户执行 `/ingest`**：扫描 `00_Inbox_原点/01_RawMaterials_原始素材` 等子目录找寻新文件。
2. **用户执行 `/ingest <path>`**：处理指定文件。

## 编译流水线 (严格执行)

### 步骤 1：深度读取

读取源文件内容。如果是非中文，在内心自动翻译并理解核心逻辑。

### 步骤 2：基底分化与页面生成 (核心)

根据内容属性进行分流：

**A. 生成主观知识 (存入 `02_Wiki_知识网络/`)**

1. 判断内容所属领域（01_Tech_软件开发 / 02_Health_身心健康 / 03_Business_商业运营 / 04_Growth_个人成长）。
2. 根据预设模板创建或更新 Markdown 笔记。
3. **必须包含严格的前置属性 (Frontmatter)**：

```yaml
title: "提取的核心概念或总结标题"
author: jiazikang
noteType: (经验总结/方法技巧/概念定义/阅读笔记/实践结果)
tags: [相关标签]
infoValue: (1-5评估)
infoDensity: (1-5评估)
status: 进行中
dateCreate: YYYY-MM-DD HH:mm:ss
dateUpdate: YYYY-MM-DD HH:mm:ss
```

1. 页面正文必须有 `## 关联链接` 区域，双链 `[[源文件名]]` 以确保溯源。

**B. 提取行动任务 (存入 `03_Action_执行空间/`)**

如果原文中包含明确的操作步骤、待办事项或改进计划：

1. 提取为 Dataview 可识别的任务格式：`- [ ] 任务描述 ⏫ #实践任务`。
2. 将其追加到 `03_Action_执行空间/04_Tasks_任务/` 中的相关笔记内。

### 步骤 3：更新大脑中枢 (Meta-Files)

必须更新 `02_Wiki_知识网络/` 下的三个元文件：

1. **`00_Glossary_术语表.md`**：记录原文中出现的新专业词汇定义。
2. **`00_Index_全局索引.md`**：在相应的领域标题下，添加新生成知识页面的双向链接。
3. **`00_Log_操作日志.md`**：追加一行记录：`| YYYY-MM-DD | ingest | [[源文件]] | 生成了知识 [[新笔记]]，提取了 X 个任务 |`

### 步骤 4：强制收件箱清零 (Inbox Zero)

确认上述步骤完成后，将原始文件从 `00_Inbox_原点` 移动到对应的底座目录：

- 技术框架/代码 -> `01_Materials_参考底座/01_TechDocs_技术文档/`
- 标准/规章 -> `01_Materials_参考底座/02_Standards_标准规范/`
- 生活/其他 -> `01_Materials_参考底座/03_LifeRef_生活参考/`

**绝对禁止修改源文件内部的文字。**

## 冲突处理

若提取的知识与 `02_Wiki` 已有概念矛盾（例如 ReAct 的推理顺序）：

暂停写入，向用户报告冲突节点，询问是“保留双重视角”还是“覆盖旧知”。
