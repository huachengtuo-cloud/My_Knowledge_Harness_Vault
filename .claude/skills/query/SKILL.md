# query 技能 (高层综合推理)

## 核心职责
你负责解答复杂的业务逻辑或概念辨析问题。你不能凭空捏造，必须依赖本地数据。

## 执行流水线 (Agentic Workflow)

### 步骤 1：调用工具 (Action -> Observation)
在回答用户前，**必须先静默调用 `search` 技能**，传入关键搜索词，获取底层的原始文本片段（Chunks）。

### 步骤 2：综合与溯源回答 (Reasoning)
基于 `search` 返回的片段进行推理分析。
- 逻辑梳理：将散落的信息编织成有条理的解答。
- **强制双链**：凡是引用了某个来源的信息，必须在句末使用 `[[文件名]]` 标注出处。

### 步骤 3：主动沉淀 (Synthesis)
回答完毕后，评估此回答是否具有复利价值。
如果是，主动询问用户：“💡 *本次解答综合了多处节点。是否需要将其转化为一篇新的经验总结存入 `02_Wiki_知识网络`？*”

### 步骤 4：更新wiki
解答结束后，更新wiki：
   - 更新 `02_Wiki/00_Index_全局索引.md`。
   - 在 `02_Wiki/00_Glossary_术语表.md` 记录新术语。
   - 在 `02_Wiki/00_Log_操作日志.md` 记录本次 Commit,格式为:`| YYYY-MM-DD | query | 检索了关键词 [XXX] | 回答了用户关于 YYYY 的综合提问 |`。
